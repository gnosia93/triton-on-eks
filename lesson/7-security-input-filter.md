## 입력값 필터링 ##

### 1. Guardrails AI (가장 대중적) ###
* Pydantic 스타일로 응답 구조를 정의하고 검증하는 데 강점이 있습니다.
* 특징: XML 기반의 .rail 파일을 사용하여 입력/출력의 형식을 강제합니다. (예: "응답은 반드시 JSON 형태여야 함", "특정 단어 포함 금지" 등)
* 장점: 구조화된 데이터를 다룰 때 매우 강력하며, 검증 실패 시 모델에게 다시 생성하라고 시키는 'Re-ask' 기능이 뛰어납니다.

### 2. Microsoft Presidio (개인정보 보호 특화) ###
* 마이크로소프트에서 만든 데이터 보호 라이브러리입니다. 
* 입력값 필터링 중에서도 개인정보(PII) 탐지 및 마스킹에 특화되어 있습니다.
* 특징: 이름, 전화번호, 이메일, 주소 등을 정교하게 찾아내어 XXXX로 바꾸거나 삭제합니다.
* 장점: 한국어 지원이 가능하며(Spacy 등과 연동), 보안이 중요한 기업용 챗봇을 만들 때 필수적으로 검토됩니다.

### 3. LLM Guard (보안 및 공격 방어 특화) ###
* 보안 전문가들이 만든 도구로, 프롬프트 인젝션이나 악성 코드 주입을 막는 데 집중합니다.
* 특징: 30개 이상의 스캐너(Scanner)를 제공하여 입력값이 위험한지 실시간으로 점검합니다. (예: 지시 사항 무시 공격, 선정적 표현 등)
* 장점: 가볍고 빠르며, 파이프라인 형태로 간편하게 적용할 수 있습니다.
