## RAGAS ##

RAGAS(RAG Assessment)는 RAG 파이프라인의 품질을 자동으로 평가하는 프레임워크으로, LLM-as-Judge 방식으로 검색 품질과 생성 품질을 모두 측정한다.

#### 핵심 메트릭 4가지 ####

![](https://github.com/gnosia93/agentic-ai-eks/blob/main/lesson/images/ragas.png)

* 검색 품질: Context Precision + Context Recall
* 생성 품질: Faithfulness + Answer Relevancy

#### 샘플 코드 ####
```
# scripts/run_ragas_eval.py


from ragas import evaluate
from ragas.metrics import context_precision, context_recall, faithfulness, answer_relevancy
from datasets import Dataset

# 평가 데이터 준비
eval_data = {
    "question": ["GPU OOM 해결 방법은?"],
    "answer": ["배치 크기를 줄이고 Gradient Accumulation을 사용하세요."],                              # LLM 응답
    "contexts": [["GPU OOM 해결: 배치 크기 줄이기, Mixed Precision, Activation Checkpointing"]],    # RAG 검색 문서
    "ground_truth": ["배치 크기 줄이기, Mixed Precision, Gradient Accumulation, Activation Checkpointing"]      # 사용자 제공 정답
}

dataset = Dataset.from_dict(eval_data)

result = evaluate(
    dataset,
    metrics=[context_precision, context_recall, faithfulness, answer_relevancy],
)

print(result)
# {'context_precision': 0.92, 'context_recall': 0.75, 'faithfulness': 1.0, 'answer_relevancy': 0.88}
```

실제 사용 흐름은:
```
1. 질문을 RAG 파이프라인에 넣음
2. 검색 결과(contexts)와 LLM 응답(answer)을 수집
3. 사람이 작성한 정답(ground_truth)과 함께 RAGAS에 넘김
4. RAGAS가 4가지 메트릭으로 평가
```

## CI-CD 통합 ##
프롬프트, 청킹 전략, 임베딩 모델 등을 변경할 때마다 자동으로 RAGAS 평가를 돌려서, 품질이 떨어지면 배포를 막는다. (UNIT 테스트와 유사) 
```
# .gitlab-ci.yml
stages:
  - evaluate
  - deploy

rag-quality-gate:
  stage: evaluate
  image: python:3.12
  rules:
    - changes:
        - prompts/**
        - config/chunking.yaml
        - config/embedding.yaml
  script:
    - pip install ragas datasets openai
    - python scripts/run_ragas_eval.py
    - |
      python -c "
      import json
      results = json.load(open('eval_results.json'))
      assert results['faithfulness'] >= 0.8, f'Faithfulness {results[\"faithfulness\"]} < 0.8'
      assert results['context_recall'] >= 0.7, f'Context Recall {results[\"context_recall\"]} < 0.7'
      assert results['answer_relevancy'] >= 0.8, f'Answer Relevancy {results[\"answer_relevancy\"]} < 0.8'
      print('Quality gate passed')
      "
  artifacts:
    paths:
      - eval_results.json

deploy:
  stage: deploy
  needs: [rag-quality-gate]
  script:
    - echo "Deploying..."
  rules:
    - changes:
        - prompts/**
        - config/chunking.yaml
        - config/embedding.yaml
```
* rag-quality-gate가 실패하면 deploy 스테이지가 실행되지 않는다. rules.changes로 특정 파일 변경 시에만 트리거 된다.
* 예를들어 results['faithfulness'] >= 0.8 의 경우 → Faithfulness 점수가 0.8 이상인지 확인
  * 0.8 이상이면 → 통과, 아무 일 없음
  * 0.8 미만이면 → AssertionError 발생, CI 파이프라인 실패
